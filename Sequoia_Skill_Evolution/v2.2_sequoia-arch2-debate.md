# Sequoia Capital Investment Research Agent (Arch 2: Multi-Agent Debate)

## 1. 触发条件
当用户要求"用红杉视角评审"、"红杉怎么看这个项目"时触发。

## 2. 核心架构：多 Agent 辩论型（Multi-Agent Debate）
- **输入**：读取 `references/sequoia_thesis.md` 知识库和目标公司报告。
- **处理**：将评审拆分为三个角色：
  - **红杉多头（Thesis）**：只找符合红杉投资标准的亮点。
  - **红杉空头（Anti-Thesis）**：只找触碰红杉反模式的致命缺陷（重点攻击护城河、毛利率）。
  - **合伙人（Partner）**：综合多空双方观点，做出最终决断。
- **输出格式**：辩论会议记录体（多头观点、空头观点、合伙人决断）。

## 3. 执行 SOP
1. 提取报告核心事实。
2. 模拟"红杉多头"发言。
3. 模拟"红杉空头"发言。
4. 模拟"合伙人"做出最终决断。
